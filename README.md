## Пошаговый запуск в Telegram

1. Установить Telegram, войти в аккаунт.
2. Меню → **Создать канал** → название и описание скопировать из `identity.md` и `about.txt`.
3. Тип: **публичный** → задать `@username` из `identity.md` (проверить, что свободен).
4. Загрузить аватар (одно изображение в едином стиле — см. `identity.md`).
5. Опубликовать пост из `post-welcome.txt`.
6. Опубликовать пост из `post-pinned.txt` (контакт заказа уже **@rasklad_babushka**; при необходимости дополните оплату из `payment-appendix-TEMPLATE.txt`).
7. Открыть пост с правилами → меню поста → **Закрепить**.
8. Обсуждения: на первом этапе можно **не** включать (по ТЗ — по желанию).

## Telegram-бот (заказы и ответы клиентам)

Код: каталог `bot_app/`. Зависимости: `requirements-bot.txt`.

1. Создайте бота в [@BotFather](https://t.me/BotFather), получите `BOT_TOKEN`.
2. Узнайте свой числовой Telegram ID (например [@userinfobot](https://t.me/userinfobot)) и пропишите в `ADMIN_IDS` (через запятую, если админов несколько).
3. Скопируйте `.env.example` в `.env`, заполните `BOT_TOKEN`, `ADMIN_IDS`, `PAYMENT_DETAILS`.
4. Установите пакеты: `pip install -r requirements-bot.txt`.
5. Запуск из этой папки проекта: `python -m bot_app` (long polling). Для работы 24/7 на сервере используйте `systemd`, Docker или процесс-менеджер; в `.env` задайте абсолютный путь к SQLite при необходимости, например `DATABASE_URL=sqlite+aiosqlite:////var/lib/babushka-bot/bot.db`.

**Логика:** клиент проходит согласие 18+, вводит вопрос, выбирает 5 или 10 USD, видит реквизиты, отмечает оплату и при желании присылает комментарий к платежу. Админы получают уведомление и отправляют расклад командой `/deliver <id>` следующим сообщением (текст, фото, голос и т.д. — пересылается клиенту).

**Команды админа:** `/orders`, `/order <id>`, `/deliver <id>`, `/cancel_delivery` (выйти из режима отправки), `/cancel_order <id>`. Команды клиента: `/start`, `/help`, `/cancel`.

Если в закрепе канала указан контакт `@rasklad_babushka` как **личный аккаунт**, а бот зарегистрирован под другим именем, в описании канала и закрепе укажите ссылку на **бота** для заказов (например `https://t.me/YourBotName`).

### Автопроверка USDT TRC20 и AI-расклад

В `.env` (см. `.env.example`):

- **`AUTO_USDT_VERIFY_TRC20=true`** — после оплаты клиент присылает **TxID** (64 hex); бот запрашивает TronScan и проверяет входящий **USDT TRC20** на адрес **`TRON_USDT_RECEIVER`** не меньше суммы заказа (5 или 10). Без ручной «верификации» в блокчейне оплата не засчитывается. Публичный API может лимитироваться — при необходимости задайте **`TRONGRID_API_KEY`**.
- **`AUTO_READING_AI=true`** и **`OPENAI_API_KEY`** — после любой **подтверждённой** оплаты (и по TxID, и по ручному сценарию с комментарием) бот вызывает OpenAI, отправляет текст расклада клиенту и закрывает заказ; админам уходит уведомление. Если модель недоступна, заказ остаётся в очереди — **`/deliver`** как раньше.

Комбинации: только авто-оплата; только AI после ручного подтверждения оплаты; оба сразу. Карты/банк (не USDT TRC20) по-прежнему только вручную или через другие интеграции.

**Дополнения:** бесплатный шаг «одна карта» (см. `ENABLE_FREE_ONE_CARD`, `FREE_CARDS_PER_DAY` в `.env.example`), последние 3 ответа в кнопке «Мои 3», визуал PNG из `bot_app/assets/cards` (создаются автоматически с Pillow, можно заменить), оплата **Telegram Stars** (см. `STARS_*`), ссылки `t.me/...?start=clarify…` / `...start=order` и текст скидки `UPSELL_DISCOUNT_TEXT`.
